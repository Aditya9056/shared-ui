Documentation

Type of Notices

Before you start adding notifications to your project, you need to learn the difference between both.

There are two types of notifications for your plugins: alert and static. Click on tabs below to learn more about them:

Alert Notices are ARIA live regions are known for not working on page load, but instead need the region to be updated after page has loaded.

In other words, a change needs to happen for Alert Notices to be recognized and read by screen readers. During page load, this change needs to happen after page has finished loading, that's when you need to trigger the function that will open the alert notification.

Float Notification

A floating notice can be an alert notification because it appears to notify users about a change that happened, for example: when user successfully creates a new palette for Hustle plugin.

Inline Notification

Inline notifications can also be considered as alert notifications when is not visually part of the content but appears when user interacts with the interface, for example: when user tries to save a module without a title for it.

Static Notices are not live regions but instead part of page regular content. This also means screen readers won't be able to read out loud its content when it appears, but instead user must reach to it when navigating through content.

In other words, Static Notices is simply some visually highlighted text. Does not matter if a change happens, screen reader won't be able to read it out loud because is not a live region. Instead, users will have to access the notice as they would with regular content.

Float Notification

A floating notice can be an static notification but is not recommended.

Inline Notification

Some times designers use notifications styles to highlight some text they need users to pay more attention to or just make a point. This notices won't need to appear after a user interaction, these will simply be there as part of the content.

Actions

These are required actions for Alert Notifications to make them appear when needed.

Open Notice

Below you can find the function to show users the message you want to notify them. You can also learn more about each of its paramethers by clicking on each section below.

SUI.openNotice( noticeId, noticeMessage, noticeOptions );
noticeId required

This is the unique ID of the element serving as the alert notice container.

const noticeId = 'unique-id';
noticeMessage required

In this paramether you must include the message would like to display in the notice. Remember it must be properly wrapped on it's tag.

const noticeMessage = '<p>Hello, I am a notification.</p>';
noticeOptions

This object paramether will allow you to overwrite default settings and styles for the notice. The allowed options are: type, icon, dismiss, and autoclose.

// Declare this only once, the rest of the options
// shown below will be attached to this variable.
const noticeOptions = {};
Type

You can leave this option empty to get default (gray) notification or use: info for blue notification, success for green notification, warning for yellow notification, error for red notification or upsell for purple notification.

noticeOptions.type = 'success'; // To show a green (success) notification.
Icon

Add the name of the icon without its prefix sui-icon-. The "loader" icon will also add sui-loading state to the icon.

noticeOptions.type = 'wordpress'; // This will show WordPress icon from SUI icons font.
Dismiss

This is an object that will handle 3 different options for dismiss button: show (visibility), label and tooltip.

noticeOptions.dismiss = {};
Dismiss Button Visibility

Change this to show or hide the dismiss button.

noticeOptions.dismiss.show = true; // This will show WordPress icon from SUI icons font.
Dismiss Button Label

Dismiss button will always be an icon button but screen readers need a proper label to notify users what's the button about. This option will help you to overwrite button label.

noticeOptions.dismiss.label = 'Dismiss Label';
Dismiss Button Tooltip

There will be cases when you need a tooltip for dismiss button. Leave this option empty for no tooltip to show up.

noticeOptions.dismiss.tooltip = 'Dismiss Tooltip';
Auto Close

This is an object that will handle 2 different options: show (to enable or disable autoclose) and timeout.

Auto Close Show

This boolean variable will allow you to enable or disable autoclose ability for notices but when dismiss button is enabled this will stop working since all dismissible notices will never auto-close.

noticeOptions.autoclose.show = false; // This will disable auto-close.
Auto Close Timeout

This option will define how much time have to pass before notice closes. By default it will close after 5 seconds. Make sure to add the value in milliseconds.

noticeOptions.autoclose.timeout = 5000; // Closes 5 seconds after show animation ends.

( function( $ ) {

	const noticeId = 'unique-id';
	const noticeMessage = '<p>Hello, I am a notification.</p>';
	const noticeOptions = {};

	// Load notification once page has loaded.
	$( window ).on( 'load', function() {
		SUI.openNotice( noticeId, noticeMessage, noticeOptions );
	});
}( jQuery ) );

HTML Actions

Open Notice

Place the following attributes on an element to trigger open notice function:

  1. This attribute will open the alert notice you assign:
    data-notice-open="unique-id" required
  2. Use this attribute to type the message that's going to be displayed in the alert notice:
    data-notice-paragraph-1="First paragraph message" required
     
    NOTE: You must have in mind the following:
    • Paragraph Allowed: There are up until 4 paragraphs allowed, or if you need some special code you can replace the paragraphs attributes with a single one called data-notice-message.
    • HTML Allowed: You can print HTML content inside of the attributes in case you have some links to show or want to make text bold, or simply need an special content inside.
  3. Use this attribute if you need to replace default "info" icon, but have in mind to not use full class name only what's after "sui-icon-":
    data-notice-icon="info"
    NOTE: Spin animation for "loader" icon will be applied automatically.
  4. Some notices need a dismiss button, when this happens you can add the attribute below with true value on it:
    data-notice-dismiss="true"
  5. This attribute is linked to the previous one, when you have dismiss button enabled you can add a tooltip to it:
    data-notice-dismiss-tooltip="Click on the button to dismiss this alert"
  6. You can also change dismiss button label when you have it enabled:
    data-notice-dismiss-label="Understood"
  7. To turn on or off autoclose ability for notification, you can use the attribute below. Just use true or false as value depending the case:
    data-notice-autoclose="false"
  8. To change the time notice will remain visible before auto-closing, you can use the following attribute:
    data-notice-autoclose-timeout="5000" 
<!--
	Notification using paragraph attributes and auto-close ability
-->
<button
	data-notice-open="unique-id"
	data-notice-paragraph-1="I am a <strong>first</strong> paragraph"
	data-notice-paragraph-2="I am a <strong>second</strong> paragraph"
	data-notice-icon="info"
	data-notice-dismiss="false"
	data-notice-autoclose="true"
	data-notice-autoclose-timeout="5000"
>
	Open notice
</button>

<!--
	Notification using message attribute (instead of paragraphs) and dismiss button
-->
<button
	data-notice-open="unique-id"
	data-notice-message="<h2>Alert title</h2><p>I am a paragraph</p>"
	data-notice-icon="info"
	data-notice-dismiss="true"
	data-notice-dismiss-label="Click to close"
	data-notice-dismiss-tooltip="Dismiss"
>
	Open notice
</button>
Close Notice

To dismiss (close) an alert notice add data-notice-close="" attribute to an element inside the notice that's going to trigger the closing action.

IMPORTANT: If the attribute value remains empty, it should be placed inside alert notice to trigger correctly the action or else value should be the unique ID from the notification to close.

<button data-notice-close="">Close this notice</button>

JS Actions

Open Notice

Trigger the function below when you need to notify a message to users:

SUI.openNotice( noticeId, noticeMessage, noticeIcon, noticeAutoClose );
  1. @param noticeOptions
    This paramether will allow you to overwrite default settings and styles for the notice. 
    let noticeOptions = {};

/**
 * Overwrite type of notice.
 *
 * Allowed options:
 * - blue
 * - info
 * - green
 * - success
 * - yellow
 * - warning
 * - red
 * - error
 * - purple
 * - upsell
 *
 * Or leave it blank to get default (gray) notification.
 */
noticeOptions.type = 'info';

/**
 * Overwrite notice icon.
 *
 * Add the name of the icon without its prefix `sui-icon-`.
 * The "loader" icon will also add `sui-loading` state to the icon.
 */
noticeOptions.icon = 'wordpress';

/**
 * Dismiss button options.
 */
noticeOptions.dismiss = {};

// Set this option true if you want to display dismiss button.
noticeOptions.dismiss.show = true;

// If dismiss button is visible, you can change its label.
noticeOptions.dismiss.label = 'A new label to dismiss';

// If dismiss button is visible, you can also show a tooltip.
noticeOptions.dismiss.tooltip = 'Dismiss tooltip';

/**
 * Autoclose options.
 */
noticeOptions.autoclose = {};

// To disable this option, you just need to set it to false.
noticeOptions.autoclose.show = false;

// When autoclose is enabled but you want to change it's timing.
noticeOptions.autoclose.timeout = 5000; // Closes 5 seconds after show animation ends.
			
( function( $ ) {

	const button = $( 'button' );

	button.on( 'click', function() {

		let self          = $( this ),
			noticeId      = 'unique-id', // Do not include number sign.
			noticeMessage = 'Type message here',
			noticeOptions = {}
			;

		// Disable autoclose.
		noticeOptions.autoclose = {};
		noticeOptions.autoclose.show = false;

		SUI.openNotice( noticeId, noticeMessage, noticeOptions );

	});
}( jQuery ) );

SUI.closeNotice( 'unique-id' );
